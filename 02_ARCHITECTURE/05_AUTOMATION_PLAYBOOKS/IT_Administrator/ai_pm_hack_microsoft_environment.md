# AI-PM HACK DESIGN
## Microsoft標準ツールを使ったPM業務効率化メモ

---

# 1. 目的

本ドキュメントは、Microsoft環境（Teams / SharePoint / Power Automate 等）を前提としたプロジェクト環境において、

PM業務の負荷を軽減し、意思決定に集中するための

「AI × 自動化」による業務ハック設計メモである。

対象は以下の状況を想定する。

- Microsoft 365 環境
- Teams中心のコミュニケーション
- Backlog / WBS 等によるウォーターフォール管理
- 大企業ガバナンス環境
- IT部門はコストセンター扱い
- PM業務の工数過多

---

# 2. PM業務の本質

PM業務は以下の3つに集約できる。

```

情報整理
意思決定
共有

```

実際の作業は以下の形になる。

|作業|割合|
|---|---|
|会議|40%|
|資料作成|25%|
|進捗管理|20%|
|調整|10%|
|思考|5%|

つまり

**作業の65%以上が事務処理**

である。

この部分を自動化する。

---

# 3. PMハックの基本戦略

基本戦略は以下。

```

作業 → AI
判断 → 人間

```

つまり

PMを

```

作業型PM
↓
意思決定型PM

```

に変換する。

---

# 6. 議事録自動化

PM業務の最大負担。

### 現状

```

会議
↓
メモ
↓
議事録作成
↓
共有

```

### AI導入後

```

Teams会議
↓
簡易メモ
↓
AI要約
↓
SharePoint保存

```

テンプレ

```

ToDo
懸念
次回アクション

```

効果

```

議事録作成時間
30分 → 5分

```
