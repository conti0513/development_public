# Cross-Cloud Transaction Data Pipeline  
(AWS → GCP → External Business System)

## Overview

This project describes the design and operation of a **cross-cloud transaction data pipeline** connecting AWS, Google Cloud, and an external enterprise system.

Transaction data generated by overseas retail stores is collected via AWS and processed in Google Cloud before being delivered to an external business platform.

The architecture was designed to ensure:

- reliable data ingestion
- secure file transfer
- automated data processing
- stable integration with external systems

---

# Background

Retail store systems located in overseas locations generate transaction data that must be centralized for:

- sales management
- BI analysis
- integration with external business systems

The previous environment faced several challenges:

- unstable data transfer processes
- operational complexity
- security and access control concerns

A new architecture was designed to establish a **reliable and automated cross-cloud data pipeline**.

# AWS Environment

AWS is used as the **data ingestion layer**.

Components:

- **AWS Transfer Family**
- **Amazon S3**

Responsibilities:

- secure SFTP ingestion
- receiving transaction data from overseas stores
- storing raw transaction files

---

# GCP Environment

Google Cloud is used for **data processing and external system integration**.

Components:

- Cloud Functions
- BigQuery
- Data transformation pipelines

Responsibilities:

- data normalization
- transformation and trimming
- preparation for external delivery

---

# Data Processing

BigQuery is used for structured data processing.

Processing steps include:

```

Raw Transaction Data
↓
Data Cleaning
↓
Column Trimming
↓
Data Formatting
↓
Export Dataset

```

Key goals:

- standardize transaction format
- remove unnecessary columns
- prepare data for downstream systems

---

# External System Integration

Processed data is delivered to an external enterprise system via SFTP.

Key requirements:

- static outbound IP
- secure file transfer
- automated scheduled delivery

This ensures reliable system-to-system integration.
