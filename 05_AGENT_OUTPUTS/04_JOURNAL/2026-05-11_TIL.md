## 2. Projects Designed Yesterday

### ip-design-tool
* **Pain point:** Network engineers manage IP addresses in Excel. Manual overlap detection. Error-prone.
* **Solution:** Web app (Next.js + TypeScript + Tailwind) for multi-site subnet management.
* **Core features:**
  * Subnet input with site name, purpose (LAN/WAN/management), redundancy flag
  * DHCP range toggle — auto-calculates range from subnet size
  * Overlap detection — warns on conflicting assignments
  * Export to CSV/Excel
* **Target users:** MSPs and SIers managing multi-site network designs.

### jtc-task-cli
* **Pain point:** Info flows through Teams / Email / verbal / SharePoint. Tasks fall through the gaps. Excel tracking is too costly to update.
* **Solution:** PowerShell CLI. 1-line input. Zero external deps. Windows-native. Local-only.
* **Core commands:**
  ```
  add   → capture task in one line
  list  → see all open tasks
  find  → keyword filter
  soon  → tasks due within N days
  done  → mark complete
  open  → launch Teams link in browser
  ```
* **Why not Trello:** JTC blocks external SaaS. PS is zero-install, offline, and fully customizable.
* **Error design:** Codes E001–E007. Does not crash. Recovers from broken JSON automatically.
* **Dev split:** CC (home) → logic and architecture. Copilot (JTC) → minor fixes only.

## 3. Design Pattern That Worked

> Spend 30 min designing with CC before touching a keyboard for code.

| Step | Action |
|---|---|
| 1. Pain point | State the real problem in plain words |
| 2. Data model | Draft the core struct / schema |
| 3. CLI / API shape | Sketch what the interface looks like |
| 4. Constraints | List what cannot be used (SaaS, internet, etc.) |
| 5. README | Write it as if explaining to a new engineer |

This pattern produced two production-candidate tools in one session.
