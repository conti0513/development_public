# テスト用システム疎通確認アーキテクチャ

## 1. 概要
本ドキュメントは、疎通確認（TEST）を目的とした標準的なスケーラブル・ウェブ・アーキテクチャの設計図である。シニアクラウドアーキテクトとして、迅速な展開と高い信頼性を重視した構成を提案する。

## 2. コンポーネント構成
- **Network**: Amazon VPC (2 Public Subnets, 2 Private Subnets)
- **Compute**: AWS Fargate (ECS cluster)
- **Load Balancer**: Application Load Balancer (ALB)
- **Database**: Amazon RDS for PostgreSQL (Multi-AZ)
- **Storage**: Amazon S3

## 3. 設計方針
1. **高可用性**: 2つのAZ（アベイラビリティゾーン）にリソースを分散し、SPOFを排除する。
2. **セキュリティ**: パブリックサブネットにはALBのみを配置し、アプリケーションサーバーとデータベースはプライベートサブネットに隔離する。
3. **拡張性**: 負荷に応じたオートスケーリングを有効化し、コスト効率を最適化する。

## 4. 総評
本構成は、小規模なPoCから商用サービスまで対応可能な汎用的なブループリントである。疎通確認完了後、IaC (Terraform/CloudFormation) による自動化フェーズへ移行することを推奨する。
