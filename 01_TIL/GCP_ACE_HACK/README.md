## 🧩 毒・薬思考（コア・エッセンス）

| 毒（アンチパターン） | 薬（マネージド正解） | ハックの急所 |
| --- | --- | --- |
| **Default SA (Editor)** | **Dedicated SA (Creator)** | 最小権限。削除権限は「毒」。 |
| **7TBデータをBQロード** | **External Tables (GCS)** | ロード時間はコストの無駄。 |
| **既存Nodeの直接改造** | **New Node Pool 追加** | GKEのスペック変更は「増築」が定石。 |
| **プロジェクト内環境混在** | **Project Isolation** | 権限・コスト・事故の隔離。 |
| **手動ルート設定(VPC間)** | **VPC Network Peering** | SDNによる自動ルート解決。 |

## 📌 現場リーダーの「寄り添い」原則

* **最小運用負荷**: 自分が楽をするのではない。組織のトータルコストを最小化する。
* **マネージド優先**: サーバーのお守りを捨て、ビジネス価値に集中する。
* **コストの透明性**: `Billing Export` + `Looker Studio` で経営層に「安心」を売る。
* **HA設計**: 単一ゾーンは「毒」。`Multi-Zone` はコスパ最強の保険。
